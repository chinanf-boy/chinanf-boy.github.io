*这本周的螃蟹行情*是在在 Github 上公开的。如果你在本周的问题中发现任何错误，请提交 PR。

Rust 社区的更新

新闻和博客文章

  • LLVM：缩小差距 - Rust 和 C / C ++之间的跨语言 LTO。
  • Weld：使用 Rust 和 LLVM 将 numpy，scikit 和 pandas 加速多达 100 倍。
  • 声明式内存管理。
  • 泛型新类型：一种解决 orphan 规则的方法。
  • Async 构建器。
  • Adventures in motion control：顶层基础架构。
  • 使用 Apache Mynewt 和 Embedded Rust 在 STM32 L476 上，构建 NB-IoT GPS 跟踪器。
  • 导航到 Rust OSS 社区。

Rust Core 的更新

在上周合并了，共有 278 个拉取请求

  • 更加一致地推断 const
  • 解析泛型参数中的 assoc 类型范围，并提供自定义诊断
  • 更新到 LLVM 9.0.0
  • rustbuild：箱子 doc 文件的 Copy 次数更少
  • 发出的 lint 无法到达时，指向原始 span
  • 在 generator witnesses 中，记录较少的调整类型，避免 MIR 构造中的 spurious drops
  • 更多ObligationForest改进
  • 用state_for_location更换成DataflowResultsCursor
  • 以正确的顺序加载 proc 宏元数据
  • 各个ObligationForest改进
  • 收缩SubregionOrigin
  • 保存分析：处理函数/方法时，嵌套 typeck 表
  • 增加cmp::{min_by, min_by_key, max_by, max_by_key}
  • 稳定化param_attrs，在 Rust 1.39.0 中
  • 确保内联常数不可变
  • 增强了跨平台编译 doctest 的能力
  • 添加 std_detect::detect::features()API，该 API 返回 target-features 的迭代器，并拒绝 feature-detection 宏中的不稳定 features

最终评论期

RFC 文档

  • [处理：推迟] cargo 版本。
  • [处理：关闭] 基于项目的示例，cargo 项目。

跟踪问题和 PR

  • [处理：合并] 稳定化某些位置上的宏。
  • [处理：合并] 稳定化 option::as_deref 和 option::as_deref_mut。
  • [处理：合并] 稳定化 mem::take (mem_take)。
  • [处理：合并: ] RFC 2008 的跟踪问题：面向 Future 的枚举/结构，带有 #[non_exhaustive] 属性。
  • [处理：合并] 支持 adts 的 repr(simd) 包含一个数组字段。
  • [处理：合并] 语法: 支持现代属性语法，在 meta matcher。
  • [处理：合并] 将 \r\n -> \n，位置在 include_str! 宏。
  • [处理：关闭] 退步: 无法确定属性宏test的 resolution。
