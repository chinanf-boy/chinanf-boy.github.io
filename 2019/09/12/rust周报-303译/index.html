Rust Core 的更新

在上周合并了 303 个请求

  • 在测试 vxWorks 时，支持静态和动态链接模式
  • 给出，参数的计数模式不匹配的变体
  • AST passes 使用 hygiene
  • 来自 proc 宏的，账号文档注释，没有 spans
  • 在无法访问的调用中，对函数名称，减少错误提示范围
  • 提前路径解析
  • or-patterns：统一在 AST＆Fix / Cleanup 解析中，使用PatKind::Or
  • 允许在 compiletest 中，检查 run-pass 执行输出
  • Rust 2018：把 NLL 迁移模式 定为重要错误
  • 为（某些）移动跟踪，扩展 Polonius fact 生成
  • polonius：完成初始化计算
  • libc：删除 WASI Core API
  • 使用 wasi 箱子，到 Core API
  • 使用 unicode-xid 箱子，而不是 libcore
  • 添加Result::cloned{，_err和Result::copied{，_err}
  • 稳定化 bind_by_move_pattern_guards 在 Rust 1.39.0
  • 稳定化 checked_duration_since 为 1.38.0
  • 稳定化 pin_into_inner 在 1.39.0
  • Rev::rposition从错误的一端，算起
  • 覆盖StepBy::{try_fold, try_rfold}
  • 添加 Iterator 比较方法，也就是采用一个 comparison 函数
  • 添加方法，可以从bool转换到Option<T>
  • Cargo：将--all重命名为--workspace
