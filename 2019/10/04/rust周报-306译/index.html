呼吁参与

总是想为开源项目做出贡献，但不知道从哪里开始？每周我们都会强调一些来自 Rust 社区的任务，供您挑选并开始！

其中一些任务也可能会有指导者，请访问任务页，了解更多信息。

  • Vector：更新stdin源码使用线程，而不是使用 tokio 版本。Vector 是一种高性能的可观察性的数据路由器。
  • mundane：启用所有功能的 docs.rs 文档构建。Mundane 是由 BoringSSL 支持的 Rust 密码库。
  • rustfm-scrobble：Hacktoberfest 的问题。rustfm-scrobble 是 Rust 中的 Last.fm scrobble API 库。
  • Tokamak 正在寻找维护者。Tokamak 是 Atom 的 Rust IDE。
  • Bastion 寻找维护者。Bastion 是 Rust 应用程序的容错运行时

如果您是 Rust 项目的所有者并且正在寻找贡献者，请提交任务这里。

Rust Core 的更新

在上周合并了，共有 278 个拉取请求

  • 安息吧，AST borrowck（借用检查员）（2012-2019）
  • 修复在 ICE 期间，打印查询堆栈时的双重 panic
  • or-模式：将PatKind/PatternKind::Or推到 HIR＆HAIR 的顶层
  • 修复 format macro expansions spans ，用宏生成
  • 删除“需要类型注释（type annotation needed）”错误的全面沉默
  • 在应该 panic 但不要 panic 的测试中，加入消息
  • 滥用 ADT 时，指向定义
  • 指向返回类型时，考虑 tail 表达式
  • obligation errors 会在可能的情况下，指出未完成的绑定
  • 修复 类型别名中的隐式特征(impl trait in type aliases)的一致性检查
  • 传播局部的types.err，进一步避免虚假的击倒错误
  • check_match：用const A: i32 = 3改进let A = 2;的诊断
  • 当期望()的条件语句时，指向封闭的 match
  • 为泛型Graph和mir::Body们，添加周期检测器
  • 添加对 SIMD 类型，插入和提取（types, insert, 和 extract）的 const-eval 支持
  • 实现基于数据流的 const 验证
  • 优化try_eval_bits，避免布局查询
  • 更好的ObligationForest改进
  • 线程本地存储的，更具解释性的 panic 消息
  • 稳定化str::len，[T]::len和str::as_bytes作为 const fn
  • 保留impl<T> From<!> for T
  • 从slice::Iter（Mut）::try_fold中删除手动展开
  • 编译器内置：实现 bcmp
  • cargo：改进--quiet的测试输出
