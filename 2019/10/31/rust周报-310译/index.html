Rust Core 的更新

在上周合并了共有 347 个拉取请求

  • 不要对完全无法展开impl Trait类型的，ICE
  • 消除intersect_opt
  • Derive Rustc{En,De}codable for TokenStream
  • 在 const 泛型中，禁止非structural_match类型
  • 将 crate store 变成一个 resolver 输出
  • 稍微简化 chalk-engine
  • 使用启发式方法，去建议分配工作
  • 为了某些义务，指向相关类型
  • Lockless LintStore
  • 删除不必要的 trait bounds 和派生
  • 更改 untagged_unions ，以不允许带有 drop 的 union 字段
  • miri：为 Memory 添加write_bytes方法，就能进行 bounds-检查和支持迭代器
  • 可安全分发的对象
  • Fix: WASI sleep impl
  • 稳定：const_constructor
  • 稳定：Option::flatten
  • 稳定：#[non_exhaustive]（RFC 2008）
  • 使is_power_of_two成为一个 const 函数
  • 为数组添加，按值 iterator
  • 添加 ：Cow::{is_borrowed，is_owned}()
  • 添加 ： [_]::{as_ptr_range, as_mut_ptr_range}()
  • 添加 ：{String，Vec}::into_raw_parts()
  • 添加 ：matches!( $expr, $pat ) -> bool宏
  • 解放在write_bytes上的ExactSizeIterator绑定
  • rustdoc：-Z选项转发到 rustc
