Rust Core 的更新

在上周合并了，共有 338 个拉取请求

  • 链接可执行文件时，仅添加清理程序运行时
  • LLVM / wasm：修复了 ret 合法化和 sjlj 之间的冲突
  • LLVM / wasm：恢复每个备忘录(memop)的存储默认值
  • 当遇到链式操作时，使用 heuristics 从不良的 turbofish 中恢复
  • 使访 visit projection iterative
  • 确定 re-export collection 的确定性
  • 重复数据闭包类型错误
  • 根据运算符的优先级，支持类型不匹配问题的修复
  • 语法：在没有类型建议的情况下，改进参数
  • Polonius：interning 时使用 fx hasher
  • 提取 async fn 生成器的期望返回类型
  • async/await：改善未发送错误
  • const-prop：尝试执行多态性提升(polymorphic promoted)的 MIR 时，修复 ICE
  • const-prop：正确处理无法传播的 locals
  • 还有更多ObligationForest改善
  • 避免chain()在find_constraint_paths_between_regions()
  • 优化 integral 模式匹配
  • proc_macro API：公开macro_rules hygiene
  • 弃用#![plugin]和#[plugin_registrar]
  • 元数据：一些箱子加载清理
  • 解引用 non-Copy 原始指针时，不进行 ICE
  • 修复，泛型数据流可视化中的 zebra-striping
  • 不要将 zero-sized 的数组的借用，标记为间接可变
  • 拒绝不在父级 impl 中，对 items 专用化
  • 为 raw_dylib 添加 feature gate
  • 稳定化，多处的，宏
  • 语法：在meta匹配器中，支持现代 attribute 语法
  • 采用PlaceBuilder，避免大量slice → vec → slice转换
  • 用try_fold代替try_for_each，减少编译时间
  • 稳定化UdpSocket::peer_addr
  • 稳定化Option::as_deref和Option::as_deref_mut
  • 稳定化todo!(..)巨集
  • hashbrown：添加RustcVacantEntry::insert_entry
  • BTreeSet intersection，is_subset和difference优化
  • Impl Clone::clone_from for LinkedList
  • 内联 {min，max}_value，即使在调试版本中
  • cargo：支持命名 profile（RFC 2678）
  • cargo：在存档中，禁止保留 mtimes
  • rustup：清除错误消息
  • rustbuild：使所有 alt 构建器，都生成启用并行的编译器
