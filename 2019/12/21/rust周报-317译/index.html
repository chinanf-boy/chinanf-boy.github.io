Rust Core 的更新

共有 223 个拉取请求在上周合并

  • revert never 类型的稳定化（sorry – llogiq）
  • 在一个 feature flag 背后，启用在 constants 的loop和while
  • 使 transparent enums 更普通
  • 确保在泛型 ZST 常量的主体，所导致执行错误，为重大错误
  • 改进诊断和代码，以彻底消除空 matches
  • 不要在 unnamed future 上，ICE
  • 从NamedMatch::MatchedSeq除掉DelimSpan
  • 优化shallow_resolve_changed
  • 添加 ExactSizeIterator bound 到 返回类型
  • fix -Z print-type-sizes 对 zero-sized 字段的处理
  • 追踪在-Z self-profile的 polonius
  • fix constant propagation for scalar pairs
  • fix macro by example code 的 unused_parens 触发
  • rustc：允许在 global trait select/eval 缓存中，使用非空的 ParamEnv
  • 删除相同的 array move MIR passes
  • chalk：消除深度传递
  • chalk：truncating 一个目标时，不 truncate 环境
  • 在 vectored I/O 中，使用第一个非空缓冲区
  • 在 Pin trait 实现中，使用 deref target
  • 改进代码生成 for starts_with('<literal char>')
  • 优化 bool 的Ord trait 实现
  • 在OsStr上内联一些常用方法
  • LinkedList：drop panics 时，drop 剩余物品
  • VecDeque：在 destructor panic 中，drop 剩余物品
  • 稳定化Result::map_or
  • 为没有格式参数的消息，添加一条单独的 path
  • remove NodeState::{Waiting,Done}
  • match VecDeque::extend to Vec::extend_desugared
  • 稳定化core::panic module
  • From<NonZero*> impls for 更广泛的NonZero类型
  • 添加 str::strip_prefix 和 str::strip_suffix
  • cargo：在[target.'cfg(debug_assertions)'.dependencies]以及类似的上，发出错误
  • rustup：改善 preinstalled rust 消息
  • docs.rs：如果无法箱子构建失败，则修复 panic viewing 源代码
