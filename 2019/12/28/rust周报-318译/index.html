你好，欢迎再来一期*本周的螃蟹行情*！Rust是一种追求三位一体的系统语言：安全性、并发性和速度。这是它的进展和社区的每周总结。有什么要说的吗？推特@ThisWeekInRust或向我们发送请求。 想参与吗？我们热爱贡献.

*这本周的螃蟹行情*是在在 Github 上公开的。如果你在本周的问题中发现任何错误，请提交 PR。

Rust Core 的更新

共有 334 个拉取请求在上周合并

  • format the world
  • 重构：expr 和 stmt 的 解析 + 升级 recovery
  • 添加一个 raw “address of” 运算符
  • 改善无效分配(invalid assignment)的诊断
  • 使用结构化建议，来消除方法调用的歧义
  • 修复：对 Drop impls 过于严格的检查
  • 保存 LTO 导入信息，并在尝试重复使用 build products 时，进行检查
  • 合并 ast::Mutability和mir::Mutability
  • 合并 TraitItem和ImplItem intoAssocItem`
  • 指示未推断类型的，类型参数的来源
  • 在一个 arena 上，分配 HIR “p1/4”
  • 在 const eval 中，添加更简单的入口点，提供常见用法
  • chalk：修复，coinductive 的不合理
  • chalk：将 ids 移入“type”家族
  • const prop 应该完成，向‘用户定义变量’的传播
  • miri：使用 Result 返回类型，支持 main 函数
  • 实现 LineWriter::write_vectored
  • 将 PartialEq和Eq添加到Cursor
  • 让 ptr::slice_from_raw_parts 为 const fn（在 feature flag 后面）
  • 稳定化：std::{rc,sync}::Weak::{weak_count, strong_count}
  • stdarch：使用更多的simd_*内部函数
  • futures.rs：让AtomicWaker::new() 为 const fn
  • cargo：修复，复写的备用 registry token
  • rustup：在 override 文件中，支持本地工具链名称
