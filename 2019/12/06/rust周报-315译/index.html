Rust Core 的更新

共有 264 个拉取请求在上周合并

  • 在 1.41.0 中，稳定化嵌套的 self receivers
  • 为 const function evaluations，添加 memoization
  • 将 crc 和 crypto 添加到 arm 上的 target feature 白名单
  • sanitizers 的条件编译
  • 添加对 sanitizers 重覆盖的支持，并跟踪未初始化内存的来源
  • async fn 的恢复，在完成后
  • 在类型参数需要Copy限定时，使用结构化建议
  • diagnostic 输出的各种调整
  • 为未实现的 trait，添加版本不匹配帮助消息
  • AST address-of
  • or-pattern 有用性检查的初始实现
  • polonius：使用占位符借用的非法子集关系错误
  • 修复函数签名中预测，导致的不透明类型
  • 简化内存分类
  • 删除 mir predecessors 缓存中的内部可变性
  • rustc：将调试信息从 LocalDecl 和 UpvarDecl 移到，专职的 VarDebugInfo 中
  • 为consts 和statics，创建 promoted MIR 片段
  • alloc：添加 new_zeroed()版本，例如 new_uninit()
  • impl TrustedLen for vec::Drain
  • 原子操作的 as_mut_ptr
  • 为 MaybeUninit 实现 Debug
  • libc：添加对 solaris/illumos 的共享内存操作的支持
  • cargo：稳定化 profile-overrides
  • rustup：添加工具链安装的 –allow-downgrade 选项
  • docs.rs：如果正在使用多个 crate-types，则要正确匹配库
