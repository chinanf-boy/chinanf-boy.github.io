Rust Core 更新

350 个请求在上周合并

  • InstCombine：不要将&mut *x优化为x
  • 添加-Z span-debug，简化 proc 宏的调试
  • 避免设置错误的 obligation，导致关联类型的 span 不匹配
  • 更小心点，generators 的ty::Error使用
  • 完成：尝试使用SmallVec或Box for stalled_on
  • impl AsRef<[T]> for vec::IntoIter<T>
  • chalk：准备首发
  • 简单default()，用为Default::default()
  • 稳定化：std::io::Buf{Reader, Writer}::capacity
  • 添加-关联的 consts MIN/MAX for Wrapping<Int>
  • 取消升级 Duration::from_secs
  • 编译器内置：为 LVI 手动修补 ret 指令
  • cargo：添加环境变量，以标识 二进制文件和 crate 名称
  • cargo：允许没有 dll 后缀的 Windows dylibs
  • cargo：传递相对路径到Workspace::new，给出更好的错误信息
  • cargo：不要在苹果平台上，哈希化可执行文件名
  • cargo：支持{prefix}和{lowerprefix} markers，在config.json dl 字段
  • cargo：如果在 git URL 中使用 hash，则发出警告
  • cargo：在 resolutions 之间，重置 lockfile 信息
  • crates.io：修复问题：crates.io 允许加号介入 crate 名称
  • docs.rs：打印“fail to build”的箱子回溯
  • rustfmt：在可见性 modifier 和 item name 之间，选取注释
  • rustfmt：保留 inner 和 outer block 文档注释中的 Markdown 换行符
  • rustfmt：使用 rewrite buffer 来确定注释是否应该在新行上
  • rustfmt: feat：条件性允许 stable/beta 上的不稳定 opts
