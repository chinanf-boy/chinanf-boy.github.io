Rust Core 的更新

共有 354 个拉取请求在上周合并

  • 启用 AVR 作为一个 Tier 3 target upstream
  • 为 x86_64-fortanix-unknown-sgx 启用 LVI 强化
  • 添加-Z span-debug，以便更轻松地调试 proc 宏
  • 添加方法，可以让一个 null-终止符 的 Vec<u8> 转为 一个CString
  • drop elaboration 后，检查常量中的 live drops
  • 显示FnMut错误内，有关‘捕获变量’的信息
  • 当 obligation 包含错误时，不要创建 impl candidates
  • 强制 unwind invariants
  • 解释，由于方法调用涉及self，而发生的 move 错误
  • 修复#[thread_local] statics 作为asm!符号操作
  • 修复：trait alias inherent impl 解析
  • 简化的default()，转为Default::default()
  • 正确处理 assembler 警告
  • 在递归 ADT 上，提供间接的结构式建议
  • 提供转换数值运算 LHS，而不是 unwrapping RHS 的建议
  • 验证一个类型是否具有结构相等性（structural equality）
  • 使存在的关联类型（existential associated types）与 variance Invariant 关联
  • 建议在注释中，包括未使用的 asm 参数，以避免错误
  • 在内部 doc 链接解析（doc link resolution）中，支持 proc 宏
  • 跟踪 - 方法调用中的函数范围(span of function)，并在#[track_caller]使用它
  • 在剩下的 rustc 箱子里，使用min_specialization
  • 使用 shorthand linker strip 参数，以支持 MacOS
  • expand：expansion-time lints 的更精确位置
  • 扩展对 HermitCore 的网络支持
  • 为 Miri，修复 caller_location intrinsic
  • 不正确的 ctypes：规范化返回类型和透明结构
  • 在结构匹配检查期间（structural match checking），标准化 adt 字段
  • resolve：不建议从我们在 resolving 的同一模块中导入
  • structure_match：non-structural-match 类型的闭包
  • chalk：添加 FnOnce trait，并为 Function 类型提供 impl
  • chalk：Rust IR 中的 model 函数 ABI
  • chalk：递归求解器 factoring 和 privacy
  • chalk：重构 ProgramClauseData，以删除隐含变体
  • chalk：添加Unsize trait 的实现
  • miri：避免重复跟踪当前位置三次
  • 移除 RawVec::reserve_in_place
  • 稳定化：Option::zip
  • 稳定化：vec::Drain::as_slice
  • impl AsRef<[T]> for vec::IntoIter<T>
  • std：在 wasm32 上，启用 atomic.fence emission
  • stdarch：修复 x86 extract_epi{8,16} 函数
  • 实现新的 gdb/lldb pretty-printers
  • cargo：添加环境变量，以标识二进制文件和箱子名称
  • cargo：允许直接将一个 ’registry index url‘ 传递给cargo install
  • cargo：修复不与--target=HOST 一起运行的 doctests
  • cargo：在config.json dl 字段，支持{prefix}和{lowerprefix} markers
  • crates.io：允许配置应用程序的域名
  • crates.io：修饰符/高亮显示语法：禁用积极的空格删减
  • doc：如果 impl block 具有关联的常量，则使其可折叠
  • docs.rs：为’上传的文档’加压缩
  • docs.rs：限制所提供文件的大小
  • clippy：宏使用建议
  • clippy：let_and_return：避免 “does not live long enough” 错误
  • rustfmt：在可见性 modifier 和 item name 之间，取用 comments
