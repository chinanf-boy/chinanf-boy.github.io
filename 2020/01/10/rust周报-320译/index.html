Rust Core 的更新

共有 207 个拉取请求在上周合并

  • 使用函数属性“frame-pointer” 代替 “ no-frame-pointer-elim”
  • 解析器：减少错误处理机制的多样性
  • 在一个 arena 上分配 HIR p4/4
  • 改善一些Drop相关的错误消息
  • 为proc_macro_server添加符号规范化
  • 建议在第一个参数为self时，调用方法
  • 实现 uncommon_codepoints lint
  • 性能：不要递归到，不需要 normalizing(规范化) 的类型
  • 出于性能原因，还原“Add IntoFuture trait and await support”
  • 确保我们在 MIR 内联过程中，处理 projections
  • miri：更新 panic 机制，以配合#[track_caller]变化
  • 修复在 const 执行期间，涉及Instance.ty调用的 ICE
  • 不再晋升 non-pattern const 函数
  • 展开 ARM EHABI 时，更新屏障缓存(barrier chahe)
  • 添加：Iterator::try_find
  • cargo：修复 CARGO_TARGET_triple_LINKER 环境变量
  • crates.io：在箱子搜索中，允许多个关键字

批准的 RFC

对 Rust 的更改跟随 RustRFC（征求意见）流程。以下是本周批准实施的 RFC：

  • RFC 2795：提议用于 formatting 宏的隐式命名参数。
  • RFC 2835：宣布安全传输项目组。
