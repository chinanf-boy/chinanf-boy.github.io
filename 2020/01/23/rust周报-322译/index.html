Rust Core 的更新

共有 270 个拉取请求在上周合并

  • 数组重复表达式的 lengths 在 MIR building time，必须是 monomorphic 的
  • 确保所有上游的 泛型，都从 Rust dylibs 重新导出
  • const 泛型参数类型内，禁止省略 lifetimes
  • 不要对格式错误的 suggestion spans，进行 ICE
  • 将 ZST 验证从整数验证中解开，并将其普及到所有 zsts
  • 不要试图force_ptr，那些指向 zsts 的指针
  • 性能：急切地将文本转换为 const
  • 通过更正更新 editor_builtins，以修复 aarch64 Windows 的 128 位整数余数
  • 处理 drop shims 的递归实例化
  • 添加无法访问的传播 mir 优化过程
  • 将 LLVM rebase 到 9.0.1
  • 不要在 bounds 不一致的 items 上，执行 const 传播
  • 不要在非 32 位 x86 MSVC 上，对 f32 cmath 函数使用 f64 shims
  • 稳定化：slice 模式
  • 从 pad_integral exit 时，重置 Formatter 标志
  • 优化 Unicode 数据集的大小/速度
  • 稳定化：Condvar::{wait_while，wait_timeout_while}
  • 稳定化：ManuallyDrop::take
  • 让iter::Empty<T> Send和Sync for 任何T
  • 实现DebugStruct::non_exhaustive
  • 实现Cursor for 链接列表（RFC＃2570）
