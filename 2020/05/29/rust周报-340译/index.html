Rust Core 的更新

359 个拉取请求在上周合并

  • 更新到 LLVM 10
  • llvm：向用户公开微小的代码模型
  • 启用 ARM TME（事务性存储器扩展）
  • 实现新asm!语法 from RFC＃2850
  • 始终为#![no_builtins]生成 object code
  • 在检查 tokens 是否“相等”之前，先中断 tokens
  • 当函数实际上具有“self”参数时，发出更好的诊断
  • 在表达式(expression)，模式(pattern)和语句位置(statement positions)中，稳定化 fn-like proc 宏
  • 采用once_cell箱子，而不是自定义数据结构
  • 简单的 NRVO
  • 删除 ReScope
  • 在结构匹配检查（structural match checking）过程中，详尽检查ty::Kind
  • 将 borrow-of-packed-field 不安全检查，移出循环
  • 在 liveness checking 期间，修复InlineAsmOperand表达式被访问两次
  • chalk：清理箱子结构，并为 SLG/递归 solvers 添加功能
  • 检查由 generator 捕获的 non-Send/Sync upvars
  • 支持FnDef和 无参数闭包之间的 coercion（强制转换），反之亦然
  • 更懒惰的，normalization of constants
  • miri：在 Windows 上，为动态符号（dynamic symbols）准备 Dlsym 系统
  • 在mem::Discriminant<T>中，采用T的判别式（discriminant）类型，而不是u64
  • 修复 generator transform 中的判别式类型
  • impl From<Cow> for Box，Rc和Arc
  • 减少size_of<HashMap>的另一种尝试
  • 非空的，初始Vec大小改为 4，而不是 1
  • 使std::char函数和常数与char关联
  • 稳定化：saturating_abs和saturating_neg
  • 添加 len和slice_from_raw_parts 到 NonNull<[T]>
  • 添加 fast-path 优化 到Ipv4Addr::fmt
  • impl Ord for proc_macro::LineColumn
  • cargo：在更新之前，尝试安装确切的版本
  • cargo：自动更新patch，如果无法进行更新，则会提供更好的错误
  • cargo：添加选项，以剥离二进制文件
  • rustfmt：从父目录合并 configs
  • rustfmt：模块解析失败时，改进错误消息
  • rustfmt：在宏定义中，解析‘逗号分隔’的分支
