Rust Core 的更新

共有 367 个拉取请求在上周合并

  • 不要在 “mir-opt-level=0” 上，运行各种 MIR 优化
  • 在box_region中，用 generator resume 参数 替换 thread_local
  • fix -Zast-json：输出正确的 JSON 形式
  • 允许librustc_data_structures/profiling.rs的 wasm32 编译
  • proc_macro::is_available()
  • proc_macro：稳定化 Span::resolved_at和Span::located_at
  • 尝试通过删除exports_all_green，来恢复性能
  • chalk：使用FxHashMap/FxHashSet，并为元组添加格式正确的条件句
  • ConstProp：使用一个BitSet<Local>，代替IndexVec<Local, bool>
  • 在 lowering 后，修复 while (let) 表达式的 span
  • Miri Frame：使用mir::Location去表示，函数中的位置
  • 添加BinaryHeap::retain
  • 新增一个函数：将Box<T>转为Box<[T]>
  • 为 linked list Cursor和CursorMut，添加缺失Send和Sync impls
  • 为NonZero整数类型，实现BitOr和BitOrAssign
  • 稳定化：alloc_layout_extras的最常见子集
  • 稳定化：Span::mixed_site
  • 稳定化：BTreeMap::remove_entry
  • futures：引入ready_chunks适配器
  • 向后移植到 0.1：避免FuturesUnordered::poll_next内的 starvation(饿死)
  • futures：添加AsyncWriteExt::write_all_vectored实用程序
  • hashbrown：future-proof （specialization code）专有代码
  • hashbrown：删除 specialization 的不合理使用
  • cargo：修复在 workspace(工作区) 中，resolve不匹配的 warning(警告)
  • cargo：添加resolver选择项，加入新功能解析器
  • rustdoc：用 JSON 解析，替换 big JS dict
