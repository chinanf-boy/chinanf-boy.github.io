Rust Core 的更新

共有 375 个拉取请求在上周合并

  • 在 float-to-int 转换中，定义 UB（未定义行为），以覆盖边缘
  • 改善 Apple 平台的 bitcode 生成
  • 不要强迫 rustc 为 LTO 构建，进行代码生成
  • 对 LLVM bitcode 发出 sections 时，像 Windows 一样正确处理 UEFI targets
  • 防止深度递归代码的编译器，堆栈溢出
  • 解决：稍微放松新的绑定歧义（binding disambiguation），修复性能退化
  • 升级 chalk，并使用 chalk-solve/chalk-ir/chalk-rust-ir
  • 报告无法移动提升的 MIR 中的错误
  • 简化 tcx.alloc_map API
  • 仅在类型是已知时，才建议删除最后一个表达式中的分号
  • 在推断变量上，跳过尝试 - 运行coerce_unsized
  • 统一所有快照（snapshot）类型的撤消日志（undo log）
  • 降低在check_match中，TypedArena的创建
  • 收缩 LocalDecl
  • 添加：remove_current_as_list至LinkedList的CursorMut
  • 添加：Arc::{incr，decr}_strong_count
  • 添加 Option 以强制展开表（Force Unwind Tables）
  • 让 BTreeMap::new和BTreeSet::new const
  • Btreemap iter 交织在一起
  • 添加：core::future::{pending，ready}
  • futures：重构 - 以减少不安全和重复代码的数量
  • cargo ：更新 LTO calculations 中的断言
  • cargo ：尝试从 http.debug 中，删除 secrets
  • cargo features：允许 activated_features_unverified 传达不存在的信息
