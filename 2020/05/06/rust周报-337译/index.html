Rust Core 的更新

共有 372 个拉取请求在上周合并

  • 实施 RFC＃2523，#[cfg(version(..))]
  • 让每个查询的缓存，将结果存储在 arenas 上面
  • 避免为每个查询，重复代码
  • 禁止dyn Trait in patterns
  • 修复：autoderef 过程中的错误参数
  • 建议into代替try_into，如果可能使用 int 类型的话
  • 调整rustc_resolve的一些建议
  • 添加消息：因名称空间错误的解析失败
  • 由?导致的错误 —— .into()上的指向返回类型
  • 在 tail exprs 中，选择建议;或是 drop 借用的行为
  • 在涉及关联类型的 type mismatch 上，建议约束
  • 最小化coerce_borrowed_pointer的参数
  • 删除一些Vec分配，以提高性能
  • 在 const-checking 中，无条件允许Unreachable terminators
  • 在 const-checking 中，无条件允许Downcast projections
  • 将 Scalars 的 MIR 常量传播，添加到函数调用参数(function call arguments)中
  • Miri：放开(unleash)所有的 feature gates
  • 使用现有框架，进行反向数据流分析
  • 添加：Read/Write::can_read/write_vectored
  • 添加：RefCell::take
  • slice::fill： 采用T，代替泛型参数
  • Vec drop和truncate：使用 raw slice *mut [T]时，就 drop
  • hashbrown：标记 RawTable::par_iter 为 unsafe
