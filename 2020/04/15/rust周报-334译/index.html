Rust Core 的更新

共有 367 个拉取请求在上周合并

  • 在extern "Rust" { ... }内，支持函数的#[track_caller]
  • 处理impl Trait—— 这里的Trait具有 missing bounds 的 assoc type
  • 在函数转换检查过程中，标准化函数签名
  • 不要丢失或重新排列，用户提供的 linker 参数
  • 在更多情况下，去建议 move —— 闭包和 async 代码块
  • 移除unused_braces的误报
  • use a SmallVec for Cache::predecessors
  • 加快路径搜索 with find_library_crate
  • 在 arena 上，分配一些查询结果(query results)
  • 添加io::Write::write_all_vectored
  • 详细的 panic 消息, for Vec函数
  • 小调整ToOwned::clone_into
  • 移除Ord bound，因它一直困扰着drain_filter
  • 在Fuse实现中，直接匹配 options
  • 用Option保险丝，实现Chain
  • 重新编排BTreeMap::into_iter，让它去匹配range_mut
  • 第一个BTreeMap提案，调整
  • 添加or_insert_with_key，到HashMap/BTreeMap的Entry
  • hashbrown：将 or_insert_with_key 添加到 HashMap 的 Entry
  • arch：添加更多 ARM SIMD 内部函数
  • cargo：添加cargo tree命令
