Rust Core 的更新

共有 394 个拉取请求在上周合并

  • 不要尝试获取，已错误的 trait impls 的’祖先’(ancestors)
  • 只跳过 foreign(外部) unstable traits 的 impls
  • 不要将()，分配到! MIR locals
  • 一些Symbol相关改进
  • use ArrayVec in SparseBitSet
  • 将SymbolName::name更改为一个&str
  • 强制让 static symbol，有序
  • 减少在 const eval 中，大量的 interning 和layout_of calls
  • 添加 Arguments::as_str()
  • impl Index<RangeFrom> for CStr
  • 向 raw（和 NonNull）slices，添加（unchecked）索引方法
  • 让一些Option方法，变为 const
  • 在 range iter impls 中，更自由地用step_unchecked
  • 添加core::task::ready!宏
  • backtrace：在 Miri 上，使用 noop 后端
  • stdarch：更新和改进 wasm32 SIMD 内部函数
  • stdarch：实现 AVX512f 浮点数比较
  • stdarch：更详细地构筑所有 x86 rustc_args_required_const内在函数
  • 使unreachable_unchecked 变为 const fn
  • cargo：修复重命名目录中，build scripts 的’新鲜度’检查
  • crates.io：使用安全的 RNG，生成 API 令牌，并存储哈希值
  • 将 Ayu 主题添加到 rustdoc
  • clippy：unnecessary_sort_by：如果 key borrows，避免进行 lint
