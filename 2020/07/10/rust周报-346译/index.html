来自 Rust Core 的更新

308 个请求在上周合并

  • 添加format_args_capture feature
  • 不要为#[target_feature]函数，实现 Fn* tratis
  • 修复了 wasm32，因一个 NodeJS 版本碰撞而损坏的问题
  • 控制 crates 之间，macro_rules! tokens 的一致性
  • 实现slice_strip feature
  • 让 likely和unlikely const 化，由 feature const_unlikely 看守
  • 利用unlikely，优化 checked_ops 的 fast path
  • 提供有关重复 lang item 错误的详细信息。
  • 去除TypeckTables::empty(None)，并将 hir_owner 设为 non-optional。
  • 从 Arc::try_unwrap 中，删除不必要的 release
  • 将所有外来SourceFile，序列化（serialize）到 proc-宏 crate metadata
  • 稳定化：#[track_caller].
  • 如果可用的话，使用 WASM 的 saturating casts
  • 使用Span，在 or-patterns 中，标识不可访问的子模式
  • 更新 rust-lang/llvm-项目子模块，以涵盖最近合并的 AVR 修复代码
  • mir opt：通过 SimplifyArmIdentity pass 修复 mis-optimization 和其他 issues
  • 添加.collect() —— 让Box<str>到String
  • impl From<char> forString
  • linker：生成 ELFs 时，默认创建GNU_EH_FRAME header
  • 解决：不允许 通过闭包/异步代码块，执行带标签的中断/继续（break/continue）
  • 船载 rust analyzer
  • chalk：添加 type outlives 目标
  • chalk：允许打印 lifetime 占位符
  • chalk：支持 ADT
  • hashbrown：添加 RawTable::erase 和 remove
  • hashbrown：暴露 RawTable::try_with_capacity
  • hashbrown：提高 RawIter 的可重用性
  • libc：添加在 Android 上缺少的一堆常量和函数
  • libc：添加更多 WASI libc 定义。
  • libc：声明seekdir和telldir for WASI。
  • stdarch：修复或整数比较
  • cargo：编写 GNU tar 文件，支持长名称。
  • crates.io：使用 default 分支别名，而不是“master”
  • clippy：添加了限制 lint —— pattern-type-mismatch
  • clippy：给出Option::map_or(_else) 建议到 if let Some { y } else { x }
  • rustfmt：不要在参数上，重复 const 关键字
  • rustfmt：不要删除 extern fn items 上的 fn headers（例如 async）
  • rustfmt：在 trait where 子句和 open block 之间，获取注释
