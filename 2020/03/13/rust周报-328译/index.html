Rust Core 的更新

共有 304 个拉取请求在上周合并

  • 实现关联类型 Defaults（RFC＃2532）
  • 在数据流期间，生成器进行 discriminant 时，不要去bug它
  • 性能：写 json 错误/警告时，缓冲 stderr
  • 像平常一样，由check_mod_attrs消耗 mark attributes
  • stash API：移除 panic，以修复 ICE
  • chalk：在 rustc 中，更改构建所需的
  • 调整 Miri value visitor，和文档注释布局组件
  • miri：让机器规范化 AllocID
  • 多个main声明的失败
  • 不要实例化这么多的drop_in_place副本
  • 切换到 discriminant 后，将其他变体标记为未初始化
  • 枚举变体若没有 drop glue，就跳过Drop终止符
  • 当用户析构函数崩溃时，审核Drop impls 中， liballoc 泄漏
  • 将原始模块，添加到 libcore
  • 放松 str::get_unchecked 前提条件，以允许空切片
  • fix：align_to_mut的混叠违规
  • 在“泄漏” RefCell 借用上添加方法，作为具有原始引用生命周期的引用
  • 稳定化：boxed_slice_try_from
  • BTreeMap导航更安全，更快捷
  • const mem::forget
  • crates.io：启用箱子排序-按最新添加的
  • rustlings：添加 clippy lints
