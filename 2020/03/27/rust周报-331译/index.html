Rust Core 的更新

共有 380 个拉取请求在上周合并

  • 在 async/await lowering 中，使用生成器恢复参数（内核异步！）
  • async：更小，更正确的生成器代码生成
  • 为 sound specialization subset 实现一个功能
  • traits 中的#[track_caller]
  • 添加#[rustc_layout(debug)]
  • parser：for<'a> |...| body闭包上的恢复
  • 解决：根据隐私错误，打印导入链
  • 解决：不要对 proc 宏定义的 visibilities，解析两次
  • ast：压缩AttrId，从usize到u32
  • 在关联类型中，修复 const 参数的类型
  • 修改了 span-to-lines 转换，在DUMMY_SP上生成一个空的 vec
  • rustc：在TypeckTables中，用LocalDefId代替DefId
  • 更新 mir 内联成本
  • 在单态化操作（monomorphizing）之后，处理ConstKind::Unresolved
  • perf（dep_graph）：在 number reads 较小时，避免分配一个 set
  • 达到 macro expansion 递归限制时，不要 unwind
  • miri：检测 UB（未定义行为）：copy/write_bytes溢出
  • 允许计算一个 point 后面的 layout
  • 使std::sync::Arc与 ThreadSanitizer 兼容
  • proc_macro_harness：使用 item header spans 来纠正错误
  • 为MaybeUninit，实现zeroed和uninitialized
  • return NonZeroU64 from ThreadId::as_u64
  • BTreeMap：删除 shared root
  • hashbrown：使用NonNull用于 Bucket 指针

本周引用句

Rust 很有趣，因为在某种意义上它真是又硬又笨重。然而，只因为*它所要成为的，恰好是如此的坚硬和笨重*。想法上，即便不牺牲任何控制权，也可以使某些东西变得更简洁、更可读、更方便，这种例子一直都有。而，有时候，某些事情就是困难的，就是不方便的，归根结底，它的质地就是那么硬。

与其他语言相比，硬语言在不需要的时候通常是笨拙的，（甚至可以说），就不应该是“简单的”。

– brundolf on Hacker News
