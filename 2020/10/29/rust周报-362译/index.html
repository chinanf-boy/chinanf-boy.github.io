Updates from Rust Core

共有 400 个拉取请求在上周合并

  • 调整 if let 建议，更自由，且不要 ICE
  • 减少在’match arms have incompatible types’ 错误中的 diagram mess
  • 调整 match arm 分号移除建议，为 futures 做准备
  • 解释被推断的闭包返回类型，是哪里的
  • 重写 collect_tokens 实现，去使用一个偏平化的 buffer
  • 修复：trait solving ICEs
  • 停止’const’中，union field 的 promoting
  • 确保 statics 类型，是有人居住的（比如空的 enum 是没人居住）
  • rustc_mir: 跟踪在 SourceScopeData的 inlined 被调用者
  • 对 const value interning 优化，for ZST types
  • 在解析期间，只计算 visibilities（可见度） 一次
  • mir-opt: 禁用 MatchBranchSimplification
  • implement TryFrom between NonZero types
  • 添加： Pin::static_ref, static_mut
  • 支持，自定义 allocators in Box
  • hashbrown: 通过一个 allocator，对 RawTable, HashSet and HashMap 参数化
  • rustdoc: 大大改善给小移动设备屏幕的显示
  • clippy: 添加 linter 给 for loop，单个元素
  • clippy: 添加 lint for &mut Mutex::lock
  • clippy: 添加 new lint for 并没有执行 Drop 得 ManuallyDrop 值
  • clippy: lint 不必要的 int-to-int and float-to-float casts
